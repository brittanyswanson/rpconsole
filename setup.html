Development

PyEnv

You can run multiple versions of python on your machine, but to manage them, I recommend install pyenv. Here's a step-by-step guide to getting this up and going on mac:

  • pyenv setup

You'll want to be sure to have the right version of Python installed (3.7), so after pyenv is installed, run the following command to install Python.

  • pyenv install 3.7.2

Code

  • Github
    • Make a Github account here
    • Install GitHub Desktop and log into your account on it
    • Follow these steps to clone the bot code to your computer (wherever you want it)
    • RP Assistant GitHub
  • Set Python Version
    • Open Terminal and navigate to where you stored the copy of code on your machine. (use cd [name_of_directory] to go inside directories)
      • example: cd /Users/bswanson/projects/rp_assistant
    • Execute the command to set the python version to the version we want
      • pyenv local 3.7.2
  • Virtual Environment
    • Always a good idea to do this so you know what libraries you're using here. It keeps it separate from other code you might be working on.
    • pip is a package manager for Python. You'll use it a lot if you want to install libraries, modules, or packages (aren't they all the same thing?)
      • Are you already running pip? Let's check!
        • python -m pip --version
      • No? Let's go download pip. Download get-pip.py
      • Using Terminal, go to the folder where you downloaded pip to (remember to use cd!
      • Run this command in Terminal: python get-pip.py
      • To update, use this command: python -m pip install -U pip
    • Install virtualenv by using this command: pip install virtualenv
    • Set up your virtual environment by navigating to where your code is stored (again with the cd!)
    • This creates a directory called venv/ where all your packages are installed. Execute: virtualenv venv
